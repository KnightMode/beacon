02 Retrieve

Evidence, not vibes.

Full-text, vectors, and the call graph fan out in parallel; an agent runs follow-up tools to fill the gaps, then reranks. Pick a question and watch it run.

03 Answer

Grounded, or silent.

The answer is built only from retrieved code, streamed into the thread, cited to file:line. No evidence? It says so — it never invents.

Verified in verifySlackSignature — it recomputes the HMAC-SHA256 of v0:{ts}:{body} and rejects stale timestamps [1].

[1] workers/slack-bot/src/signature.ts:21-44

04 Review

Reviews that have read the codebase.

Paste a PR URL — or react with an emoji — and get a streamed review informed by the whole index, not just the diff.

DR

@beacon review github.com/acme/api/pull/318

05 Fix & ship

From issue to open PR.

Describe the problem in a thread, react 🚀, and Beacon proposes the edits and opens the pull request for you.

⬡

fix: cap D1 batch lookups at 100 bound params

opened by beacon[bot] · 2 files · ready for review

06 Triage

A red build becomes a cited diagnosis.

When a GitHub Actions run fails, Beacon posts a diagnosis to your channel — grounded in the failing logs and the diff. React 🚀 for a fix PR. Transient flakes get a re-run nudge.

CI failed on main — likely the 100-param D1 cap in hydrateContent [1]. React 🚀 to open a fix.

↺ …and that diagnosis is the next question. The loop closes.

Why not the tools you already have

"Don't we already have AI for this?"

Claude or ChatGPT?

A chat window that has never seen your codebase — plausible is the dangerous kind of wrong.

Beacon acts only on retrieved code, cited to file:line — and doesn't stop at answering.

Cursor or Copilot?

Brilliant — for whoever has the editor open. The rest of the loop lives in Slack.

Beacon works the loop for the whole team: answer, review, fix, ship.

Code search?

Search returns files, not answers.

Beacon traces the call graph and explains why.

A SaaS code bot?

Your repos on their servers, plus a per-seat invoice.

Beacon runs on your Cloudflare. MIT-licensed. Zero servers.
